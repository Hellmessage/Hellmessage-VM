# HVM 文档总入口

本目录是 HVM 项目的全部设计文档与决策沉淀, 分两个版本并存:

| 目录 | 内容 | 用法 |
|---|---|---|
| [v1/](v1/) | **现状描述** — 按当前代码逻辑重构后的设计文档 (架构 / 模块 / 协议 / 约束) | 想了解"项目长什么样"看这里 |
| [v2/](v2/) | **下一步行动** — 2026-05-03 全项目深审产出的 45 项 TODO 清单 | 想知道"接下来做什么"看这里 |

## 快速入口

### 我想了解项目

→ [v1/README.md](v1/README.md) — v1 文档索引, 推荐阅读顺序

### 我想知道下一步做什么

→ [v2/README.md](v2/README.md) — v2 TODO 总览, 按 P0/P1/P2 分级

### 我想看约束

→ 项目根目录 [../CLAUDE.md](../CLAUDE.md) — 唯一权威约束源
- 与 v1/ 描述冲突时, 以 CLAUDE.md 为准
- v2/ 中标记的"修 CLAUDE.md"项是已识别的待修订点

## 版本约定

- **v1** 是"现状快照", 描述代码现在长什么样
- **v2** 是"未来动作", 描述要把代码改成什么样
- v2 项完成后:
  - 勾掉对应 TODO
  - v1 同步更新现状描述
  - 涉及约束变化时同步 CLAUDE.md
- v2 全部清完后, v2 文档可滚动归并到 `CHANGELOG.md`(届时新建 v3 起新一轮审查)

## 治理

- 文档与代码漂移是常见问题, v1 重构每季度 review 一次, 维持准确度
- 新设计决策 → 直接落 v1(或起 v3 文档目录)
- 已知漂移 → 立即在 v2/04-doc-drift.md 登记, 而不是放任
